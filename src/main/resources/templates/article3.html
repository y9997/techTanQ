はじめに

Python開発において、仮想環境を構築することはとても重要です。
仮想環境を利用することで、プロジェクトごとの依存関係を分離し、環境の管理をより適切に行うことができます。

本記事では、初学者向けにPythonの仮想環境について基本から構築方法までまとめました。
本記事に沿って構築まで行うことで、仮想環境の基本知識とスキルが身につく構成になっていますので、ぜひ最後までご覧ください。

基本概要

仮想環境とは

「仮想環境」と聞くと、少し難しそうな印象を受けるかもしれません。
でもイメージを掴めれば大丈夫です。

仮想環境とは、１台のPCの中に複数の独立した作業スペースを作る仕組みのことです。
例えば、macOSのPCでWindowsを構築するようなことです。
これは、Macという物理的な環境の中にWindowsという別の環境を仮想的に構築している状態です。

Pythonの仮想環境においても基本的な考え方は同じです。
なので、Pythonにおける仮想環境とは、さまざまなバージョンのPythonやライブラリなどの環境をプロジェクトごとに１つのPC上で構築することになります。

仮想環境は違いに独立しているため、プロジェクトごとに異なるライブラリを使用することや、チーム開発の差異簡単に環境を統一できるなどのメリットがあります。

一方で、Pythonには「グローバル環境」と呼ばれるものもあります。
Pythonをインストールした直後に仮想環境を構築しなかった場合に使用される、PC全体で共有される環境です。
仮想環境とは違い、ライブラリや設定などをすべてのプロジェクトが共有することになるため、プロジェクト同士が干渉しやすくなります。
そのため、Pythonの開発では仮想環境の利用が推奨されています。

仮想環境のしくみ

Pythonの仮想環境は環境変数とディレクトリ構造を使って環境を作り、activateスクリプトにより目的の環境を有効化することで実現しています。
以下のステップで仮想環境が作られます。

0.ローカルにPythonをインストール
仮想環境は、ローカルにインストールされたPython実行ファイルをもとに作成されます。
このPythonを本記事では「システムPython」と呼びます。
1.仮想環境ディレクトリの作成
仮想環境のベースとなるディレクトリを作成すると、その中に独立したPython実行環境とライブラリ群が構築されます。

ディレクトリ構造(macOS)

ポイント
myenv/bin/pythonにはシステムPytonのコピーまたはリンクが配置されます。
仮想環境は独立しているように見えますが、実際にはシステムPythonに依存しています。

2.仮想環境の有効化(activate)
仮想環境を有効かすると、環境変数PATHが書き換えられ、Pythonやpipの実行対象が仮想環境内のものに切り替わります。

PATHの変化

以上がPython仮想環境の仕組みの概要です。
実際には、こうした仕組みが裏で動いていることで、プロジェクトごとに環境を分離し、安定した開発が可能になります。
2章では、実際に仮想環境を構築する手順をmacOSとwindowsに分けて紹介します。

仮想環境ツールの種類

Pythonの仮想環境を管理するツールにはさまざまな種類があり、プロジェクトの目的や用途に応じて使い分けることが重要です。

venv
Python3.3以降に標準搭載されている仮想環境ツールで、公式推奨されています。
昨日もシンプルで軽量なので、初心者の方におすすめです。
次章ではこの仮想環境ツールでの構築方法を紹介します。

virtualenv
venvの全身で、多機能や柔軟性が高いツールです。
Python2でも使用することができます。

conda
データサイエンスや機械学習で使用されている環境ツールです。
Pythonだけでなく、RやCライブラリも管理可能な点や、パッケージ管理と仮想環境管理が一体化しているところが特徴です。

pipenv
pipとvirtualenvを統合したツールです。
チーム開発や依存管理が重要な場合に適しています。

poetry
近年人気が高まっているツールです。
仮想環境の作成、依存関係の管理、パッケージの公開までを一括で扱えるのが特徴です。
ライブラリの配布やPyPI公開を視野に入れる開発者におすすめです。

仮想環境構築方法

この章では、macOSとWindowsそれぞれの仮想環境構築手順をまとめました。

macOSの場合

1.Pythonがインストールされているか確認

python3 --version

　　※バージョンが確認できない場合は以下のコマンドでインストールしてください。
brew install python

2.仮想環境を作成


　　作業ディレクトリに移動後、以下のコマンドを実行
python3 -m venv myenv

　　※本記事では仮想環境名をmyenvにていますが、自由に設定可能

3.仮想環境の有効化


source myenv/bin/activate

　　有効化するとプロンプトが(myenv)のように変化します。

4.仮想環境の終了


deactivate

windowsの場合

1.Pythonのインストール確認

python --version
もしくは
py --version

　　※表示されない場合は公式サイトからインストール
https://www.python.org/downloads/windows/

2.仮想環境の作成


python -m venv myenv

3.仮想環境の有効化


myenv\Scripts\activate

　　プロンプトが(myenv)のように変化します。

4.仮想環境の終了


deactivate
